How it works

The honest version.

When you send a message, claude.ai’s own response stream ends with a quota event — the same data its UI uses internally. Headroom simply makes it visible.

  1. 1

    Observe

    A content script wraps the page’s fetch to watch responses. The original stream passes through untouched.

  2. 2

    Extract

    The parser reads the message_limit event — and discards everything else. Your conversations are never stored or logged.

  3. 3

    Store locally

    Snapshots live on your device only, with a 90-day cap. No backend exists.

  4. 4

    Render

    A Shadow-DOM badge shows both windows, color-coded, with a live countdown to reset.

Privacy

Nothing leaves
your browser. Ever.

  • No backend — there is no server to send data to
  • No telemetry, analytics, or crash reporting
  • No account, no sign-up, no API keys
  • Conversation content is never stored — quota numbers only
  • Host access limited to claude.ai, nothing else
  • Source-available now — audit the code yourself
